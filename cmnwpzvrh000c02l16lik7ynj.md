### [IT 鐵人賽] Azure Data Service - Day 01 - Intro

#### Azure Data Service - Day 01 - Introduction

現在 AI 當道，但是在 AI 的背後其實還是由 Big Data 所組成的，俗話說的好：「沒有 Data 就沒有 AI」。

當然在這現代分工非常明確與細緻的時代，我們沒有必要全部都自己來，往大了說可以專精 Data，AI 的部分就交給別人來做，反之亦然。甚至你都不需要懂，也是有直接現成好的服務提供使用。

---

![](https://cdn.hashnode.com/res/hashnode/image/upload/v1776070396300/68ec465f-97f9-42a7-9dc5-b974ac8c5b54.png)

Microsoft Azure 身為一個正統雲端服務提供商，當然也提供了許多相關類型的服務，我在這邊統一稱為 Azure Data Service。

從最基本的數據儲存開始，例如：

* Azure SQL Database
* Azure SQL Database for MySQL
* Azure SQL Database for MariaDB
* 等等

到儲存極大數據與不同類型資料的服務：

* Azure Cosmos DB
* Azure Data Lake
* 等等

有了數據以後就可以開始使用這些數據進行機器學習的訓練，Azure 所提供的相關服務有：

* Azure Machine Learning Studio
* HDInsight
* Azure Databricks
* 等等

> Azure Machine Learning Studio 跟 HDInsight 看起來已經有大大的主題是這一塊了，所以我到時候應該不會著墨太多。

另外還有剛剛提到的已經建立好的服務直接提拱給你使用：Cognitive Service，他提供了許多面向類型：

* 辨識
  可聰明地識別、標示及 控管圖片的影像處理演算法。
* 語音
  將語音轉換成文字、使用語音來驗證，或將說話者 辨識新增至您的應用程式。
* 知識
  對應複雜資訊和資料，以解決 智慧建議和語意搜尋等工作。
* 搜尋
  將 Bing 搜尋 API 新增至您的應用程式，並充分利用 以單一 API 呼叫合併數十億個網頁、影像、影片和新聞的能力。
* 語言
  讓您的應用程式能夠透過預先建置的指令碼處理自然語言、 評估情感，以及了解如何辨識使用者想要的項目。

我會首先詳細介紹與使用 Cognitive Service，再來的話回到 Data 相關，最後就是關於機器學習的部分；如果還有多的時間的話，還可以介紹一下擁有 GPU 的虛擬機器類型。

在這 30 天內，透過撰寫不同服務類型的文章，期許我自己可以從一個 Data、AI 雙白，轉變成小白。
