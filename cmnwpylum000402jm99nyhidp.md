回想當初為什麼會踏入使用 Visual Studio 的行列

還依稀記得高中二年級的時候，每周會有一兩堂的選修課，那時就選修了程式設計，課堂上教的是 Visual Basic 6.0，練習寫了一個小計算機後，就開始翹課了..，第一次用到 Visual Basic 6.0 覺得實在很奇妙，有些東西只要拉一拉，然後再寫點程式就可以 WORK

之後大學選了資訊工程，真正的第一次接觸程式語言，學校教了 C++，一開始用的 IDE 是 Borland C++ Builder，後來有一次在學校電腦裡看到了 Visual Studio 2003，裡面包含了 Visual C++，那時覺得這個編輯器怎麼可以這麼炫，於是就開始用 Visual Studio

到後面開始視窗程式設計課程後，我還是用 C++ 做視窗程式設計，但是後來助教和同學都開始改用 C# 開始寫程式，為了可以快速的學別人 ( 複製 ) 寫的程式就開始學寫 C#，當然也就繼續用 Visual Studio 做開發，從一開始的 Visual Studio 2003 到 Visual Studio 2010 都覺得視覺效果很不錯，另外就是 IntelliSense 真的很不錯，用到後來沒有 IntelliSense 的輔助都快要不知道怎麼寫程式了

出了社會，就還是朝原來比較熟悉的方向繼續做，發現 Visual Studio 2012 有越來越多的功能，不論是輔助 C# 或是 ASP.NET 開發或是透過擴充功能撰寫一些額外程式語言，都非常的好用

Visual Studio 也整合了所有微軟的系列程式開發產品，從一開始比較有學的 MFC 到後來的 WPF、Windows Universial App 開發，以及最近的 Xamarin 都是用 Visual Studio 開發

Visual Studio 即將滿 20 個年頭，期待 Visual Studio 可以繼續幫助程式開發人員精進效率
